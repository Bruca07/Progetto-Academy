37. Cos'è un Web Container? Qual è il suo ruolo in un'applicazione JEE? Fai un esempio di
Web Container che conosci.

RISPOSTA:
Un Web Container è un programma che fa da tramite tra il browser e la tua applicazione Java.
Quando un utente apre una pagina, il Web Container riceve la richiesta, la passa al pezzo di codice giusto e rimanda la risposta al browser.
Un esempio famoso è Apache Tomcat.

38.Qual è la differenza tra un sito web statico e un'applicazione web dinamica? Fai un
esempio concreto di ciascuno.

RISPOSTA: 
Sito statico: le pagine sono già pronte, uguali per tutti. Il server le manda così come sono, senza fare nulla.
Applicazione dinamica: la pagina viene generata al momento, in base all'utente o alla situazione.

39.Cos'è un file WAR? Cosa contiene e perché è importante nel deploy di un'applicazione
Java web?

RISPOSTA: 
WAR sta per Web Application Archive. È semplicemente un file compresso, contiene le pagine HTML, CSS, JavaScript, le Servlet (il codice Java),
il file di configurazione web.xml, le librerie necessarie è importante perché per mandare in produzione la tua applicazione ti basta un solo file. Lo prendi e lo "butti dentro" Tomcat (o un altro Web Container), che lo legge e mette online la tua applicazione automaticamente.

40.Cos'è il protocollo HTTP? Descrivi brevemente cosa succede quando un browser fa una
richiesta a un server web (ciclo request-response).

RISPOSTA:
HTTP è il linguaggio che usano il browser e il server per comunicare tra loro.
Il ciclo request-response:
Digiti un URL nel browser, ad esempio www.google.com
Il browser manda una richiesta al server
Il server la elabora e manda una risposta
Il browser mostra la pagina.

41. A cosa serve il file web.xml in un'applicazione JEE? Fai almeno un esempio di
configurazione che può contenere.

RISPOSTA:
È il file di configurazione della tua applicazione web. 
Dice al Web Container come deve comportarsi con la tua applicazione.

42.In base a quanto studiato, qual è la differenza tra JEE e Jakarta EE? Perché è avvenuto
questo cambiamento di nome?

RISPOSTA:
Sono la stessa cosa, cambiata solo di nome.
Java EE era di proprietà di Oracle. Nel 2017 Oracle ha ceduto il progetto alla Eclipse Foundation, un'organizzazione open source.
Il problema è che Oracle non ha ceduto il marchio "Java", quindi non era possibile continuare a chiamarla Java EE.
La Eclipse Foundation ha quindi rinominato il tutto in Jakarta EE.
